Project 3: Responsive Business Website Final project build karna with modern design.

Project Goal

Hum ek responsive business website banayenge jiska naam hoga BrightWeb Studio. Is project mein top navbar, hero section, services section, about section, and contact CTA section hoga. Website desktop par wide and professional dikhegi, aur mobile par clean one-column layout mein adjust ho jayegi.

Is project ka purpose hai ki learners sirf CSS properties yaad na karein, balki unko real website par implement karke samjhein. Har CSS topic final project ke kisi na kisi part mein use hoga.

HTML Code Detailed Explanation

Code Detailed Explanation
<link rel="stylesheet" href="styles.css"> HTML file ko CSS file se connect karta hai. Agar yeh line missing hogi toh styling apply nahi hogi.
class="container navbar" Same element par two classes hain. container width control karega, navbar Flexbox layout apply karega.
class="nav-links" Navigation links ko target karne ke liye class. CSS se links ka gap, color, and hover style set hoga.
class="hero" Hero website ka first main section hota hai. CSS isko spacing and background dega.
class="hero-grid" Hero section ke content ko Grid layout mein arrange karega: left text, right card.
class="button" Normal link ko button design dene ke liye class.
class="services-grid" Services cards ko CSS Grid se columns mein arrange karne ke liye wrapper.
class="service-card" Each service ko card design dene ke liye class. CSS se padding, border, shadow, and hover effect milega.

CSS Code Detailed Explanation

CSS Code Detailed Explanation
* { box-sizing: border-box; } Universal selector saare elements par apply hota hai. border-box layout calculation ko easy banata hai because padding and border width ke andar count hote hain.
body Complete page ke base styles set karta hai: margin remove, font, color, background, and line-height.
a All links ko target karta hai. Underline remove hoti hai and links parent color inherit karte hain.
.container Content ko center and max readable width mein rakhta hai. Desktop par content zyada spread nahi hota, mobile par 92% width use hoti hai.
.site-header Header ko white background, border, and sticky top behavior deta hai.
.navbar Flexbox layout apply karta hai. Logo left and links right side align hote hain.
justify-content: space-between; Navbar ke first item ko left and last group ko right side push karta hai.
.nav-links Navigation links ko row mein arrange karta hai and links ke beech gap add karta hai.
.nav-links a:hover Mouse hover hone par link ka color blue ho jaata hai. Yeh transition topic ka simple use hai.
.hero Hero section ko top-bottom spacing and gradient background deta hai.
.hero-grid CSS Grid use karta hai. Left column text ke liye wide hai, right column card ke liye smaller hai.
.button Anchor link ko button jaisa design deta hai: background, white text, padding, rounded corners.
transition Hover effect ko smooth banata hai. Color and movement suddenly change nahi hote.
.hero-card, .service-card, .about-box, .contact-box Group selector hai. In sab boxes ko same card design milta hai.
.services-grid Services cards ko 3 equal columns mein arrange karta hai.
@media (max-width: 768px) Mobile and tablet screen ke liye responsive CSS rules start karta hai.
grid-template-columns: 1fr; Mobile screen par Grid ko one-column layout mein convert karta hai.

Step 4: Run and Test Output

  1. VS Code mein dono files save karo.
  2. index.html ko browser mein open karo.
  3. Desktop par navbar, hero, services, about, contact sections check karo.
  4. Browser width kam karo ya mobile view open karo.
  5. Mobile par navbar vertical and cards one-column hone chahiye.

Expected Output

Desktop par website mein top sticky header hoga. Logo left side and navigation links right side honge. Hero section mein left side heading and button hoga, right side feature card hoga. Services section mein 3 cards ek row mein dikhenge. About and contact sections white card style mein dikhenge.

Mobile par layout simple ho jayega. Navbar vertical ho jayega, hero section one-column ho jayega, and services cards one-by-one stack honge. Button hover par smooth movement and dark blue color change dikhega.

Common Mistakes

Mistake Fix
CSS apply nahi ho rahi. Check karo styles.css same folder mein hai and link tag correct hai.
Cards grid mein nahi aa rahe. Check karo services wrapper par class="services-grid" hai.
Mobile layout break ho raha. Media query section check karo.
Button hover work nahi kar raha. Check karo link par class="button" hai.
